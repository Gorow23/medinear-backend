STEP 1 — SET UP MONGODB ATLAS (FREE DATABASE)
================================================================

1. Go to: mongodb.com/cloud/atlas
2. Click "Try Free" → Sign up
3. Choose FREE tier (M0 Sandbox)
4. Select region: Mumbai (ap-south-1) or Singapore
5. Create cluster (takes 2-3 minutes)

6. Create Database User:
   - Click "Database Access" in left menu
   - Click "Add New Database User"
   - Username: medplus_admin
   - Password: (choose a strong password, SAVE IT)
   - Role: "Read and write to any database"
   - Click "Add User"

7. Allow Network Access:
   - Click "Network Access" in left menu
   - Click "Add IP Address"
   - Click "Allow Access from Anywhere" (0.0.0.0/0)
   - Click "Confirm"

8. Get Connection String:
   - Click "Database" → "Connect"
   - Choose "Drivers"
   - Copy the connection string — looks like:
     mongodb+srv://medplus_admin:<password>@cluster0.xxxxx.mongodb.net/
   - Replace <password> with your actual password
   - Add "medplus" at the end: ...mongodb.net/medplus
   - SAVE THIS STRING — you need it next

================================================================
STEP 2 — DEPLOY BACKEND TO RENDER.COM (FREE)
================================================================

Render.com hosts Node.js servers for free!

1. Go to: render.com
2. Click "Get Started" → Sign up with GitHub
   (If you don't have GitHub: create free account at github.com)

3. Push backend to GitHub:
   a. Go to github.com → New Repository
   b. Name it: medplus-backend
   c. Upload all files from the "backend" folder

4. Deploy on Render:
   a. Go to render.com dashboard
   b. Click "New" → "Web Service"
   c. Connect your GitHub account
   d. Select "medplus-backend" repository
   e. Fill in settings:
      - Name: medplus-backend
      - Runtime: Node
      - Build Command: npm install
      - Start Command: npm start
   f. Click "Create Web Service"

5. Add Environment Variables on Render:
   - In your service → click "Environment"
   - Add these variables:
     
     Key: MONGODB_URI
     Value: (your MongoDB connection string from Step 1)
     
     Key: JWT_SECRET
     Value: (any random string, e.g. "medplus_super_secret_2024_xyz")
     
     Key: FRONTEND_URL
     Value: https://medpluspharmacy.netlify.app
     
     Key: PORT
     Value: 5000

6. Your backend URL will be:
   https://medplus-backend.onrender.com
   
   Test it: https://medplus-backend.onrender.com/api/health
   Should show: { "status": "ok" }

HOW MULTI-TENANCY WORKS (NOTES)
================================================================

Every piece of data in MongoDB has a "storeId" field.
This is called the TENANT KEY.

  Products:  { storeId: "store_A_id", name: "Paracetamol" }
  Orders:    { storeId: "store_A_id", customerId: "...", ... }

When Store A logs in and views their dashboard:
  - They ONLY see products where storeId = their store ID
  - They ONLY see orders where storeId = their store ID
  - They CANNOT see Store B's data

This is how one database safely serves many stores!

API ENDPOINTS REFERENCE
================================================================

AUTH:
  POST /api/auth/register   → Create account
  POST /api/auth/login      → Login
  GET  /api/auth/me         → Get current user

STORES:
  GET  /api/stores                    → All stores
  GET  /api/stores/nearby?lat=&lng=   → Nearby stores
  GET  /api/stores/:id                → One store
  POST /api/stores                    → Register store (auth)
  PUT  /api/stores/:id                → Update store (auth)

PRODUCTS:
  GET    /api/products?storeId=xxx    → Store's products
  GET    /api/products/:id            → One product
  POST   /api/products                → Add product (store owner)
  PUT    /api/products/:id            → Update product (store owner)
  DELETE /api/products/:id            → Remove product (store owner)

ORDERS:
  POST /api/orders              → Place order (customer)
  GET  /api/orders/my           → My orders (customer)
  GET  /api/orders/store        → Store's orders (store owner)
  PUT  /api/orders/:id/status   → Update status (store owner)
